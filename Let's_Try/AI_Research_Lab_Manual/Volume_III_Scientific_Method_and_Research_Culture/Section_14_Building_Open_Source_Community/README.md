## Prompt for Deep Content Generation

```
You are a developer relations strategist, open-source community manager, 
and AI researcher writing for a solo lab founder.

Write Section 14 of "Building an AI Research Lab From Scratch" — 
a complete guide to building an open-source AI research community.

PART A — WHY OPEN SOURCE FOR A RESEARCH LAB
   - The strategic case for open source from a lab-building perspective
   - Open source as: talent pipeline, credibility signal, funding mechanism, 
     research accelerator, competitive moat (paradoxically)
   - Case studies: Hugging Face's open source flywheel, EleutherAI, Mistral
   - When NOT to open source: IP protection, competitive moats, safety concerns

PART B — GITHUB STRATEGY
   - Repository design for research projects: what makes a great ML research repo
   - README structure for a research paper's code release
   - Contribution guidelines: how to make contributing easy
   - Issue templates: bug report, feature request, paper discussion
   - GitHub Actions for ML repos: testing, formatting, automated benchmarks
   - Release strategy: versioning, changelog, release notes
   - How to get 1,000 stars in the first week after a paper release

PART C — DISCORD COMMUNITY DESIGN
   - Channel structure for an AI research community
   - Onboarding flow: how new members get introduced and oriented
   - Moderation: rules, enforcement, the "brilliant jerk" problem
   - Engagement loops: weekly paper discussions, experiment sharing, Q&A
   - Roles and leveling: contributor, researcher, core team
   - Discord bots for research communities: arXiv bot, paper discussion threads

PART D — TWITTER/X STRATEGY FOR AI RESEARCHERS
   - How top AI researchers use Twitter effectively (with examples)
   - Content cadence: paper threads, experiment updates, opinion posts
   - Paper launch threads: how to write one that gets 10K impressions
   - Building a following from 0: what works, what doesn't
   - The "value before ask" principle: how to build authority before promoting your work
   - Twitter lists for AI research: how to use them

PART E — BLOG AND TECHNICAL WRITING
   - Lab blog strategy: what to write, how often, for whom
   - Blog post types: paper explanation, tutorial, opinion, experiment report
   - Distribution: HuggingFace Blog, Towards Data Science, Substack
   - SEO for research blogs: how to get Google to send you readers
   - The "distillation" effect: explaining papers better than the original authors

PART F — DOCUMENTATION
   - Why documentation is the most underrated research output
   - What great ML library documentation looks like: PyTorch, HuggingFace Transformers
   - Tools: Sphinx, MkDocs, Docusaurus
   - Tutorials vs. API reference vs. conceptual guides
   - Interactive documentation: Jupyter notebooks, Google Colab links

PART G — BENCHMARKS AND LEADERBOARDS
   - Why creating a benchmark gives you permanent community relevance
   - How to design a fair, contamination-resistant benchmark
   - Hosting leaderboards: HuggingFace Spaces, PapersWithCode, EvalAI
   - Organizing evaluation competitions: prizes, rules, anti-gaming
   - The "Goodhart's Law in benchmarks" problem and how to mitigate it

PART H — OPEN DATASETS AND COMMUNITY GRANTS
   - Releasing datasets: HuggingFace Hub, Zenodo, academic data repositories
   - Dataset cards and documentation standards
   - Community compute grants: how to offer them and what you get in return
   - Community bounties: paying contributors to fix bugs or add features

PART I — CONTRIBUTOR ONBOARDING
   - The "first contribution" problem: how to make it achievable
   - Good first issues: what makes one actually good
   - Mentorship model: how to invest 2 hours/week in contributors who become researchers
   - Converting contributors to employees: the pipeline
   - Recognition and attribution: how to keep contributors motivated

PART J — SUSTAINABILITY
   - Open source business models: open core, consulting, sponsorship, SaaS
   - GitHub Sponsors: how to set it up, what revenue is realistic
   - Corporate sponsorship for open source AI: how to approach companies
   - Grants for open source projects: Ford Foundation, Schmidt Futures, NLnet
   - The "free rider" problem and how successful projects handle it

Format: structured markdown, community platform comparison table,
GitHub repo template, Discord channel structure diagram.
Depth target: ~15–20 pages.
```
